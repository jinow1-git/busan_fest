# 제품 요구사항 정의서 (PRD) - 부산 축제 정보 플랫폼 "축제해 부산" (Fest Busan)

본 문서는 부산광역시의 오픈 API를 활용하여 부산 지역의 다채로운 축제 정보를 사용자에게 효과적으로 제공하는 웹 사이트의 제품 요구사항을 정의합니다. 본 프로젝트는 **Next.js**와 **Tailwind CSS**를 기반으로 개발되며, **Vercel**을 통해 배포될 예정입니다.

---

## 1. 프로젝트 개요 (Project Overview)

*   **서비스명**: 축제해 부산 (Fest Busan)
*   **목적**: 부산광역시에서 개최되는 다양한 축제 정보를 한눈에 확인하고, 검색, 필터링 및 위치 기반 지도로 탐색할 수 있는 반응형 웹 플랫폼 구축.
*   **핵심 가치**: 
    *   **시각적 즐거움 (Visual Delight)**: 부산의 바다와 활기찬 에너지를 담은 감각적이고 현대적인 UI/UX 제공.
    *   **접근 편의성 (Accessibility)**: 구군별, 월별 필터링 및 상세 교통 정보를 통한 높은 탐색 편의성.
    *   **개인화 (Personalization)**: 북마크 기능을 통해 관심 축제를 나만의 리스트로 관리.
*   **데이터 출처**: 부산광역시 축제 정보 서비스 Open API
    *   `https://apis.data.go.kr/6260000/FestivalService/getFestivalKr`

---

## 2. 타겟 사용자 (Target Audience)

1.  **부산 방문 예정 여행객**: 축제 일정을 확인하고 동선 및 교통 정보를 파악하고자 하는 내·외국인 관광객.
2.  **부산 시민**: 주말이나 여가 시간에 참여할 만한 지역 축제를 쉽게 탐색하고자 하는 로컬 주민.
3.  **교통 약자**: 축제 인프라(장애인 주차장, 휠체어 접근 가능 여부 등) 편의 정보를 사전에 확인하고자 하는 사용자.

---

## 3. 기술 스택 (Tech Stack)

*   **프레임워크**: Next.js (App Router 기반)
*   **스타일링**: Tailwind CSS
*   **상태 관리 / 클라이언트 데이터**: React Context API, LocalStorage (북마크 저장용)
*   **지도 라이브러리**: Leaflet (오픈소스 지도 라이브러리, API 키 없이 즉시 지도 구현 가능) 또는 Kakao Maps API (사용자 설정 시)
*   **아이콘**: React Icons / Lucide React
*   **배포 플랫폼**: Vercel

---

## 4. 기능적 요구사항 (Functional Requirements)

### 4.1. 메인 대시보드 (Home Page)
*   **히어로 섹션 (Hero Section)**:
    *   부산의 역동성을 상징하는 비주얼 요소(그라데이션 및 고화질 배경)와 검색 바 배치.
    *   인기 축제 또는 다가오는 축제 하이라이트 슬라이더/캐러셀.
*   **축제 카드 그리드 (Festival Grid)**:
    *   각 카드는 썸네일 이미지, 제목, 개최 기간, 구군명, 간략한 태그 표시.
    *   호버 시 확장 및 미세 애니메이션 효과 적용.
*   **검색 및 필터링 시스템**:
    *   **검색**: 축제 제목 및 내용 키워드 검색.
    *   **구군 필터**: 부산시 16개 구·군별 필터링 (예: 해운대구, 수영구, 영도구 등).
    *   **날짜/월별 필터**: 사계절별 혹은 12개월별 축제 분류 탐색.
    *   **접근성 필터**: 휠체어 접근 가능, 장애인 화장실 등이 지원되는 축제 별도 보기 기능.

### 4.2. 상세 정보 보기 (Detail Page / Modal)
*   **통합 뷰포트**: 
    *   사용자의 몰입감을 높이기 위해 깔끔한 Detail Page 또는 인터랙티브 모달 지원.
*   **표시 정보**:
    *   고화질 메인 이미지.
    *   축제 타이틀, 서브타이틀, 소개글(줄바꿈 및 가독성 고려).
    *   **축제 정보 시트**: 개최 날짜, 이용 요금(무료/유료 구분), 문의처, 공식 홈페이지 링크.
    *   **교통 및 위치**: 상세 교통수단 안내(지하철, 버스 등) 및 행사장 도로명 주소.
    *   **무장애 여행 정보(Middle Size RM1)**: 편의 시설(유모차/휠체어 대여, 전용 주차장 등) 시각적 아이콘으로 안내.
    *   **위치 지도**: Leaflet 지도를 통한 행사장 핀포인트 매핑.

### 4.3. 개인화 기능: 내 축제 (Bookmarks)
*   사용자가 관심 있는 축제 카드에서 '하트' 또는 '북마크' 아이콘을 클릭하여 나만의 리스트에 저장.
*   로그인 없이 브라우저 `localStorage`를 통해 영구 보존.
*   상단 네비게이션 또는 필터에 '내 보관함' 탭을 두어 저장된 축제만 모아보기 제공.

### 4.4. 지도 모아보기 (Map Explorer)
*   부산 전체 지도 화면에서 현재 진행 중이거나 예정된 축제들을 마커로 표시.
*   마커 클릭 시 미니 정보 카드 팝업 표시 및 상세 페이지 이동 지원.

## 5. UI/UX 디자인 원칙 (UI/UX Principles)

*   **테마 및 컬러 시스템**:
    *   **Deep Ocean Blue & Neon Wave**: 부산의 깊은 바다와 야경의 네온사인을 테마로 한 그라데이션.
    *   **글래스모피즘(Glassmorphism)**: 투명하고 세련된 카드 스타일 및 모달 디자인.
    *   **다크 모드 지원**: 트렌디한 다크 모드를 기본/옵션으로 지원하여 야경 불꽃축제 느낌의 몰입형 경험 제공.
*   **인터랙션**:
    *   버튼, 카드 호버 시 부드러운 스케일 업 및 섀도우 효과.
    *   데이터 로딩 시 매끄러운 스켈레톤 UI 제공.
*   **반응형 웹**:
    *   모바일, 태블릿, 데스크톱 등 모든 디바이스 크기에 완벽하게 최적화된 Grid/Flex 레이아웃.

---

## 6. 비기능적 요구사항 (Non-functional Requirements)

*   **SEO 최적화 (SEO Best Practices)**:
    *   페이지별 고유 타이틀 및 메타 디스크립션 적용.
    *   시맨틱 HTML 구조(`header`, `main`, `section`, `article`, `footer`) 준수.
    *   상세 페이지 동적 메타데이터 적용.
*   **성능 최적화**:
    *   Next.js Image 컴포넌트를 이용한 이미지 최적화 (자동 리사이징 및 Lazy Loading).
    *   API 호출 결과 캐싱 또는 Static Generation + Revalidation(ISR) 활용으로 빠른 로딩 속도 유지.
*   **웹 접근성**:
    *   모든 이미지에 `alt` 태그 추가.
    *   시각 장애인을 위한 ARIA 어트리뷰트 제공 및 키보드 네비게이션 지원.

## 7. 단계별 개발 로드맵 (Development Roadmap)

1.  **Phase 1: 기획 및 구조 설계**
    *   PRD 작성 (완료)
    *   Next.js 프로젝트 초기 구성 및 Tailwind CSS 설정
2.  **Phase 2: UI 및 코어 컴포넌트 개발**
    *   레이아웃, 헤더, 푸터 컴포넌트 제작
    *   디자인 시스템 구축 (Color Palette, Card, Filter UI)
3.  **Phase 3: API 연동 및 데이터 관리**
    *   부산 축제 API 패칭 모듈 구현
    *   로딩 상태(Skeleton) 및 예외 처리
    *   검색, 필터, 북마크 상태 로직 구현
4.  **Phase 4: 고급 기능 통합**
    *   Leaflet 지도 연동 (축제 위치 렌더링)
    *   LocalStorage 연동 관심 축제 기능
5.  **Phase 5: 성능 테스트, SEO 보완 및 배포**
    *   LightHouse 성능 측정 및 최적화
    *   Vercel 원클릭 배포
